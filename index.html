1 Le problème

À partir d'un ISBN, récupérer une vraie couverture et pousser le taux de réussite aussi proche de 100 % que possible — y compris pour les catalogues islamiques français et arabes où les APIs grand public (OpenLibrary, Google Books) tombent à ~60–70 %. Le pipeline enchaîne 5 phases de fallback à travers 21 sources sur 11 noms d'hôtes distincts ; chaque phase ne se déclenche que si la précédente n'a pas retourné de couverture valide.

2 Pipeline — chaîne de fallback en 5 phases

Chaque phase est une course ou une boucle séquentielle ; la première couverture valide court-circuite les phases suivantes.

3 Filtres qualité — ce qui est rejeté

Filtres appliqués à chaque image candidate avant acceptation. L'ordre d'exécution va du plus rapide (URL) au plus coûteux (décodage image).

4 Stratégie anti-ban

Concurrence limitée par hôte
Sémaphore par nom d'hôte. Défaut = 2 requêtes en parallèle. Surcharges : googleapis & openlibrary = 8, m.media-amazon.com = 4, backapi.leonardo-service.com = 4.
Gigue aléatoire
Pause aléatoire avant chaque requête : 50–200 ms (HTTP simple), 80–300 ms (hôtes Cloudflare). Casse le rythme uniforme inter-requêtes.
Rotation de User-Agent
Round-robin atomique sur 6 chaînes UA bureau (Chrome / Firefox / Safari / Edge sur Mac / Windows / Linux). Un UA par requête.
Retry unique sur 429 / 503
Une seule nouvelle tentative après 600–1400 ms aléatoires — pas d'escalade exponentielle. Les échecs passent à la source suivante.
Usurpation TLS
CycleTLS + cloudscraper_go pour les sites Cloudflare JS (maison-ennour, tawhid, sana). Imite un vrai ClientHello Chrome.
Diversification des sources
21 sources distinctes sur 11 hôtes. Un blocage sur l'un (Amazon-FR, Bing) laisse 20 autres intacts — dégradation gracieuse sans interruption.

5 Cache

Cache disque à /cache/<isbn>.json. Clé = ISBN normalisé. Un hit saute tout le pipeline et répond en < 50 ms. Nettoyeur lancé toutes les 24 h.

TTL positif : 30 jours (couvertures trouvées restent en cache)  ·  TTL négatif : 7 jours (les 404 sont retentés plus tôt — les sources s'améliorent avec le temps).

6 Problèmes rencontrés & résolus

  • ✓ Fausses couvertures banques d'images — des modèles Canva apparaissaient dans les résultats Bing/Yandex. Résolu par liste de blocage de domaines + filtre ratio portrait (rejet si l/h hors 0,45–0,95).
  • ✓ Pages WooCommerce « aucun résultat » — les sites islamiques français (Tawhid, Sana) retournent une grille « vous aimerez aussi » pour les ISBN inconnus. Résolu par détection de la classe CSS search-no-results et vérification de la présence de l'ISBN sur la page produit.
  • ✓ Mauvaises éditions — même titre sous plusieurs ISBN ; la recherche image retournait la mauvaise édition. Résolu par vérification stricte que l'ISBN apparaît dans l'URL ou le texte de la page produit.
  • ✓ Cloudflare sur les librairies islamiques françaises — maison-ennour, edition-tawhid, librairie-sana bloquent les clients HTTP Go natifs. Résolu avec CycleTLS + cloudscraper_go.
  • ✓ albouraq.fr est une SPA React — le HTML rendu ne contient pas les données produit. Résolu en reverse-engineerant son endpoint REST backapi.leonardo-service.com/api/sofiadis/albouraq/articles?smart_search=<isbn>.
  • ✓ Saturation mémoire navigateur à 1 500+ ISBNs — JSZip côté client avec 1 500 images base64 atteignait 2–4 Go de heap. Résolu en déplaçant l'assemblage ZIP côté serveur ; le navigateur ne gère qu'un lien de téléchargement en streaming.

7 Chronométrage bout en bout

Latence typique par ISBN selon la phase qui trouve en premier. Un échec total (404) épuise toutes les phases séquentiellement.

Afficher les couvertures en Java Swing

Swing pur — aucun framework externe. Charge les images depuis un dossier local (ex. : le ZIP décompressé) et/ou depuis des URLs distantes.

Fichiers dans le ZIP : nommés exactement <isbn-saisi>.jpg (tirets et espaces préservés) — accès direct : new File(folder, isbn + ".jpg").

1) Lire et afficher une image locale

// CoverViewerLocal.java — javac CoverViewerLocal.java && java CoverViewerLocal /path/to/covers 9780143126393 import javax.imageio.ImageIO; import javax.swing.*; import java.awt.*; import java.awt.image.BufferedImage; import java.io.File; public class CoverViewerLocal { public static void main(String[] args) throws Exception { File folder = new File(args[0]); String isbn = args[1]; File jpg = new File(folder, isbn + ".jpg"); BufferedImage img = ImageIO.read(jpg); // reads JPEG/PNG/GIF if (img == null) throw new RuntimeException("Not an image: " + jpg); JFrame f = new JFrame("Cover - " + isbn); f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); f.add(new JLabel(new ImageIcon(img))); f.pack(); f.setLocationRelativeTo(null); f.setVisible(true); } }

2) Charger une image distante depuis une URL

// CoverViewerRemote.java — fetches over HTTP without any framework import javax.imageio.ImageIO; import javax.swing.*; import java.awt.image.BufferedImage; import java.net.HttpURLConnection; import java.net.URL; public class CoverViewerRemote { public static BufferedImage fetch(String url) throws Exception { HttpURLConnection c = (HttpURLConnection) new URL(url).openConnection(); c.setRequestProperty("User-Agent", "Mozilla/5.0"); // Amazon & co reject default UA c.setConnectTimeout(5000); c.setReadTimeout(10000); try (var in = c.getInputStream()) { return ImageIO.read(in); } } public static void main(String[] args) throws Exception { String isbn = args[0]; // e.g. "9780143126393" String url = "https://covers.openlibrary.org/b/isbn/" + isbn + "-L.jpg?default=false"; BufferedImage img = fetch(url); if (img == null) throw new RuntimeException("No cover for " + isbn); JFrame f = new JFrame("Cover - " + isbn); f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); f.add(new JLabel(new ImageIcon(img))); f.pack(); f.setLocationRelativeTo(null); f.setVisible(true); } }

3) Galerie : charger toutes les couvertures d'un dossier

// CoverGallery.java — grid view of all .jpg files in a folder import javax.imageio.ImageIO; import javax.swing.*; import java.awt.*; import java.awt.image.BufferedImage; import java.io.File; public class CoverGallery { private static JLabel makeThumb(File f, int w, int h) throws Exception { BufferedImage src = ImageIO.read(f); Image scaled = src.getScaledInstance(w, h, Image.SCALE_SMOOTH); JLabel lbl = new JLabel(new ImageIcon(scaled), JLabel.CENTER); lbl.setText(f.getName().replace(".jpg", "")); lbl.setVerticalTextPosition(JLabel.BOTTOM); lbl.setHorizontalTextPosition(JLabel.CENTER); lbl.setForeground(Color.LIGHT_GRAY); return lbl; } public static void main(String[] args) throws Exception { File folder = new File(args[0]); File[] files = folder.listFiles((d, n) -> n.toLowerCase().endsWith(".jpg")); if (files == null) files = new File[0]; JPanel grid = new JPanel(new GridLayout(0, 5, 12, 12)); grid.setBackground(Color.DARK_GRAY); grid.setBorder(BorderFactory.createEmptyBorder(12, 12, 12, 12)); for (File f : files) grid.add(makeThumb(f, 120, 180)); JFrame frame = new JFrame("Cover Gallery — " + files.length + " books"); frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); frame.add(new JScrollPane(grid)); frame.setSize(820, 700); frame.setLocationRelativeTo(null); frame.setVisible(true); } }
Conseil : exécutez chaque exemple avec javac Fichier.java && java Fichier <args> sur JDK 11+. Pour de grandes galeries, effectuez les I/O hors de l'EDT (utilisez SwingWorker) — ces extraits restent minimalistes pour la clarté.
